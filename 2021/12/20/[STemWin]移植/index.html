STemWin 移植

1. 为什么移植 STemWin

  1. 最近有一个想法，为了加速硬件/固件的调试，特别是涉及到一些通信协议以及硬件相关手板的调试等，可以以直观的方式进行调试
  2. 在一些新人不了解某些通信协议的情况下，只需要在 GUI 界面进行简单的操作就可实现特定协议通信，协议的组包及解析在底层进行，避免了不熟悉通信协议而无法展开工作
  3. 在抛弃 PC 的情况下，用于自动化测试或者老化测试

2. 怎么移植 STemWin

  1. STemWin 实质是 Middleware 中间件，所有的中间件都是有些类似的
    • 从架构来讲，中间件都是承接应用层与硬件层的(类似于 linux 驱动中的内核框架与硬件驱动之间的关系)
    • 中间件提供一些底层硬件接口给移植者，移植者需要将当前使用的硬件的相关驱动(LCD，触摸屏)等与 Middleware 提供的接口对接起来
    • 调用 Middleware 提供的 API 函数写自己的应用层代码即可
  2. STemWin 移植的简要架构如下

  1. STemWin 移植的步骤
    • 准备一个工程项目(包含 LCD 及触摸屏的底层驱动)
    • ST 官网下载 STemWin 的中间件(一般情况下 CubeFx 里面是最新的 STemWin 版本)
    • 对接 STemWin 中的硬件驱动接口
    • LCD 的显存空间的分配
    • 应用中初始化 GUI 及调用 GUI API 即可

3. 移植过程

  1. ST 官网下载 STemWin 库(使用 STemWin 5.26 版本)

    https://my.st.com/content/my_st_com/zh/products/embedded-software/mcu-mpu-embedded-software/stm32-embedded-software/stm32cube-mcu-mpu-packages/stm32cubef1.license=1595729594812.product=STM32CubeF1.version=1.8.0.html

  2. LCD 的底层驱动中的不能有 LCD 宏定义以及 LCD_Init 函数，因为这两个定义在 STemWin 中已经被使用

  3. 添加 STemWin 的源码到工程中，并添加好相应的头文件路径

    • GUIConf.c
    • GUIConf.h
    • GUIDRV_Template.c
    • LCDConf_FlexColor_Template.c
    • GUI_X.c
    • STemWin526_CM_xxx.lib

  4. 自行创建 LCDConf.h 头文件

  1. 修改 GUIConf.h 配置文件对 GUI 相应的支持项进行配置(如，支持 OS，支持触摸等等)

  2. 修改 GUIConf.c 来配置 Framebuffer 的内存空间(由于 LCD 需要较大的 Framebuffer 显存空间)，需要申请一片较大的空间用作显存的缓冲区

    • 是否使用外部 SRAM 的宏定义
    • emWin 可使用的内存大小以及块大小宏定义
    • 通过 GUI_X_Config 配置函数来自行配置申请内存空间大小以及指定内存空间中的块大小

  1. GUIDRV_Template.c 对接 LCD 底层驱动的挂接功能函数(主要是一些对 LCD 基本操作的功能函数，如打点函数，读点函数，区域填充函数等)

    • 打点函数

    • 读点函数

    • 填充长方形函数
    • 16bpp 位图绘制函数

  2. LCDConf_FlexColor_Template.c 中函数的挂接，这个文件主要是对 LCD 的驱动 IC 控制器的寄存器读写来对 LCD 控制器进行配置(可以实现 LCD 驱动 IC 控制器的初始化，LCD 扫描方式的控制，对 GRAM 数据读写的控制等等)，GUI 中间件中虽然自带了一些 LCD 的驱动程序，但是有些 LCD 的驱动程序 GUI 是不支持的，所有就需要自己实现并将 GUI 中的驱动替换掉

    • 以下这 4 个函数(LCDWriteReg, LcdWriteData, LcdWriteDataMultiple, LcdReadDataMultiple )可以删除掉，因为在硬件初始化时(初始化 LCD 驱动控制器)，已经使用自己的初始化函数对控制器进行了初始化了
    • 修改 LCD_X_Config 函数，通过函数 GUI_DEVICE_CreateAndLink 将前面 GUIDRV_Template.c 中自己实现的一些底层驱动函数挂接到这个结构体中(有函数指针指向)，GUI 中间件在需要使用时会回调这些函数指针指向的函数进行相应的操作

  1. 使用 STemWin 的时候，记得使用前一定需要开启 CRC 时钟，否则无法使用.

  2. 触摸屏相关移植

    • 触摸屏的底层初始化
    • 创建 GUI_X_Touch_Analog.c 文件，添加电阻屏或者电容屏扫描被触摸点的坐标

  3. 定时器提供时基

    • 为 emWin 提供 1ms 时基的定时器
    • 为 emWin 提供 10ms 时基轮询触摸屏是否有触摸发生

4. 移植 GUI Demo 工程

  1. 按照上面的步骤移植好 GUI Middleware
  2. 向工程中添加 GUI Demo
  3. 主程序中直接调用 GUIDEMO_Main()即可
