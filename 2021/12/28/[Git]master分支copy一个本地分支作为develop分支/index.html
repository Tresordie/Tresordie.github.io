Git中从master分支copy一个本地分支作为develop分支

|阅读量:

Git 中从 master 分支 copy 一个本地分支作为 develop 分支

1. 查看当前分支状态

  1. git branch

  2. git status

  3. 如果当前不在 master 分支，需要切换到 master 分支

    git checkout master

  4. 拉取远端仓库使当前的 master 分支更新到最新版本

    git pull

2. 基于当前 master 分支，创建一个本地分支

  1. 基于 master 分支，创建一个本地分支 test，并切换到此本地分支

    git checkout -b test

  2. 查看当前的所有分支，包含本地分支及远端分支

    git branch -a

3. 将新建本地分支 push 到远端仓库

  1. push 本地分支到远端仓库

    git push origin test

  2. 执行上面的步骤后，远端的 test 分支已经创建并被同步

4. 建立本地到远端仓库的链接(确保本地代码能提交到远端)

  1. 建立本地 test 分支到远端仓库的链接

    git branch --set-upstream-to=origin/test

  2. 如果不设置此链接，test 分支的数据会被推送到 master 分支

5. 再次拉取分支

  1. 从远端分支拉取此分支到本地

    git pull

6. 分支提交更新

  1. 添加内容到 test 分支
  2. 提交并 push 到远端 test 分支

7. master 分支更新并提交

  1. master 分支更新内容
  2. 提交到 master 分支
  3. git graph 可以查看当前分支

8. 将 test 分支 merge 到 master 分支

  1. 将当前分支切换到 master 分支

    git checkout master

  2. 拉取远端 master 分支

    git pull

  3. merge test 分支到 master 分支

    git merge test

  4. 提交并 push 到远端

    git commit -m "content"

    git push origin master

  5. 可以通过 git graph 查看 merge 结果
