[RISC-V]GD32VF103CBT6 开发环境搭建(续)

1. 基于 Windows 的 IDE 的开发坏境搭建(基于 C 语言)

  1. 首先明白 GD32VF103CBT6 这款 RISC-V 芯片的 IP Core 是由国内的芯来科技设计开发的，芯片是国内芯片公司兆易创新基于 IP Core 设计的(模式类似于 ARM)

  2. 针对 Windows 系统下开发，先到芯来科技的官网下载其针对 RISC-V 的 IDE 开发工具(基于 Eclipse 开发的)

  3. 兆易创新官网下载基于芯片的 SDK 包

  4. 下载 zadig 软件

  5. 兆易创新官网下载芯片的 DFU 工具及驱动

  6. 打开 IDE 及 SDK 的一个例程项目，里面与 RISC-V 内核相关的部分都是芯来科技编写并提供的，芯片的片上外设层的库是兆易创新编写提供的

  7. 编写固件，点个 LED 等，编辑好代码后，在左侧相应的项目上鼠标右键点击编译，编译完成后回生成.hex 或者.elf 的文件

  8. 编译成功之后，就需要对芯片进行固件的烧录了，这里个人觉得国内这个芯片做的非常好的地方是可以将芯片进入 DFU 模式；进入之后可以直接进行固件的烧录，而不需要借助任何其它的烧录工具，DFU 支持的 USB 协议，在烧录前需要安装 DFU 的驱动，且需要将芯片的驱动配置为 libusb 的驱动方式，需要做的步骤如下：

    • 安装 DFU 驱动
    • 芯片驱动安装成 libusb
    • USB cable 连接 PC 与目标板，目标板进入 DFU 模式
    • 开启 GigaDevice 的 DFU 固件烧录工具进行烧录
    • 烧录完成后，手动 reset 目标板运行程序
